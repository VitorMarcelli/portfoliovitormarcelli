# Contexto de Execução — Projeto Novo Portfólio

## 1. Objetivo deste documento
Este documento registra o contexto operacional atual do projeto, sem alterar o PRD do produto.

Ele existe para separar:
- definição do produto;
- ferramenta de execução atual;
- modelo de IA atual;
- regras operacionais da implementação.

---

## 2. Ambiente atual de execução

### Ferramenta principal
OpenCODE dentro do ambiente de trabalho atual.

### Modelo de IA atual
Codex GPT-5.3

### Observação
Este contexto poderá mudar no futuro sem exigir alteração do PRD do produto.

---

## 3. Papel do ambiente de execução
O ambiente atual será usado para:
- estruturar o projeto;
- gerar código;
- apoiar decisões de implementação;
- organizar a arquitetura inicial;
- acelerar a criação das bases do frontend;
- apoiar modelagem de dados e organização de componentes.

---

## 4. Limites do contexto atual
O ambiente atual não define o produto. Ele apenas executa o produto.

Portanto:
- decisões de produto devem seguir o PRD;
- decisões visuais devem seguir a direção criativa definida;
- qualquer solução genérica deve ser evitada;
- não se deve confundir velocidade com qualidade final.

---

## 5. Regras operacionais

### Regra 1 — O PRD é soberano
O PRD define o produto. O ambiente de execução não pode desvirtuar o escopo.

### Regra 2 — Evitar soluções genéricas
Toda implementação deve buscar alto padrão visual e estrutural.

### Regra 3 — Separação de responsabilidades
- PRD define o que o produto é;
- este documento define como estamos executando agora;
- o plano de implementação define o que fazer nesta fase.

### Regra 4 — Preparar para troca futura
Nada deve ser estruturado de forma que dependa conceitualmente do ambiente atual.

### Regra 5 — Não transformar contexto operacional em regra permanente de produto
Se a LLM, o executor ou o fluxo mudarem, o produto continua sendo o mesmo.

---

## 6. Diretrizes de execução para esta fase
O ambiente atual deve priorizar:
- base sólida;
- código limpo;
- boa organização;
- componentização;
- escalabilidade;
- facilidade de manutenção;
- preparação para evolução visual premium.

---

## 7. Critérios de qualidade desta execução
A execução atual deve:
- evitar improviso;
- manter estrutura profissional;
- gerar base limpa;
- documentar decisões;
- validar funcionamento real;
- preparar terreno para a fase visual.

---

## 8. Observação final
Este documento pode ser atualizado sempre que o ambiente operacional mudar, sem necessidade de reescrever o PRD.
